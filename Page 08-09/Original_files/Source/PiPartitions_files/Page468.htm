Raspberry Pi Super Powers — The SUDO Command

 

In any linux system, security is key.  Nothing is allowed to install or change configuration unless it's run by a super user.

 

Your account you created when you installed Debian (the one called “pi” by default), is not powerful enough to carry out these tasks.  At first this may seem like a limitation, but in fact it's a boon.  If your account can't do very much, then any malicious processes that may attempt to make changes whilst you are using your computer can't either!

 

There is always an account called “root” on any linux setup.  This account has super-duper and omnipotent powers over your system.  Never, and I can't stress this enough, never log in as root.  You don't need to.  I'm not going to explain why here as I haven't the space, but trust me on this.  Google it for more info!

 

So, what do you do when you want to install some software, or make changes that require this super power?

 

For this you need a command called “sudo”.

 

At a terminal prompt, try the following:

 

cd /

touch test

 

Did you get a permission denied error?  That's because you don't have permission to write to the root (/) of your file system.  In that example, the touch command tried to create a file called “test”.

 

Now try:

 

cd /

sudo touch test

 

It will ask for your password, and then execute the command just fine.

 

Check the file is there.  Type:

 

ls -lh test

 

And press enter.  You should see:

 

-rw-r--r-- 1 root root 0 2012-05-14 21:14 test

 

Don't worry about the bit at the start (all the -rw--r-- stuff), just look at the end.  There is our file.

 

Now get rid of it/  Type:

 

rm test

 

And press enter.

 

So what happened?  First you may have got a confirmation asking if you were sure, and then another permission denied?  Why do you think you got that?  Yup, you don't have the permission to delete it!

 

Now type:

 

sudo rm test

 

And press enter.

 

This time, you may or may not get prompted for your password.  Sudo remembers the password for a short amount of time, so it depends on how quickly you got to this point.  But to confirm the file has gone type:

 

ls -lh test

 

And press enter.

 

Nothing?  Good!  Give yourself a star.

 

So what does sudo do?

 

The sudo command says to the system; “I want to make myself an admin for a short period of time please.  And when you've done that, please execute the command I've typed.  Thank you”.  The system first checks you have permission to type the sudo command in the first place, and then asks for your password.  If all is good it goes ahead and runs the command as if you were a super-user.

 

So why is this better?  You can still issue those dangerous commands!  Well yes, but firstly the act of having to use sudo should hopefully make you think twice, and more importantly you never need to know the root password.  You just use your own.  Again, space prevents me from listing all the benefits, but you will find plenty of information here:
